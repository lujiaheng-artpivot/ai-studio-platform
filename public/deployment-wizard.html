🚀 artpivot.top 生产环境配置向导

按照以下步骤完成生产环境部署

完成进度: 0/5

1
配置 Vercel 生产环境变量

访问 Vercel 项目设置，添加以下环境变量（Environment: Production）

NEXT_PUBLIC_APP_URL=https://artpivot.top NEXT_PUBLIC_CANONICAL_HOST=artpivot.top NEXT_PUBLIC_CLERK_FRONTEND_API=modern-falcon-75.clerk.accounts.dev NANOBANANA_API_KEY=20e604c6c997931bdced6a0b12a2d05d ... (其他变量见文档)
💡 提示: 可以使用自动化脚本快速配置：
./setup-vercel-env.sh
打开 Vercel Dashboard
2
配置 Google OAuth

在 Google Cloud Console 配置 OAuth 2.0 授权

Authorized JavaScript origins:

https://artpivot.top https://www.artpivot.top https://app.artpivot.top https://modern-falcon-75.clerk.accounts.dev

Authorized redirect URIs:

https://modern-falcon-75.clerk.accounts.dev/v1/oauth_callback
⏰ 重要: 配置保存后需要等待 5-10 分钟才能生效
打开 Google Cloud Console
3
配置 Stripe + 支付宝

3.1 启用支付宝

在 Stripe Dashboard -> Settings -> Payment methods 中启用 Alipay

3.2 配置 Webhook

添加 Webhook endpoint:

https://artpivot.top/api/webhooks/stripe

监听事件:

  • checkout.session.completed
  • checkout.session.async_payment_succeeded
  • checkout.session.async_payment_failed
  • checkout.session.expired
⚠️ 重要: 复制 Webhook Signing Secret 并更新到 Vercel 环境变量 STRIPE_WEBHOOK_SECRET
打开 Stripe Dashboard
4
线上自检

验证所有配置是否正确

检查项目:

  • ✓ 健康检查 API
  • ✓ Google 登录测试
  • ✓ 支付宝支付测试
  • ✓ 账户算力检查
5
支付测试与问题排查

完成一次完整的支付流程测试

测试步骤:

  1. 访问 pricing 页面
  2. 选择 Lite 套餐（$10）
  3. 使用支付宝支付
  4. 完成支付后检查 dashboard
  5. 验证算力是否增加
💡 如果支付后未到账:
  • 检查 Stripe Events 中的 webhook 响应
  • 如果是 pending_user，重新登录一次
  • 可以在 Stripe 中手动 resend event
查看 Stripe Events

🎉 配置完成！

你的网站已经可以正常使用了
